API Reference

The ExampleSDK API uses REST conventions with JSON request and response bodies. All endpoints require authentication via Bearer token.

Authentication

Include your API key in the Authorization header:

curl -H "Authorization: Bearer sk_live_xxxxx" https://api.example.com/v1/users

Generating API Keys

Navigate to Settings → API Keys in your dashboard. Each key has configurable scopes and expiration dates. Production keys should have minimal permissions.

Users

Create a User

import examplesdk

client = examplesdk.Client(api_key="sk_live_xxxxx")
user = client.users.create(
    email="alice@example.com",
    name="Alice Johnson",
    role="admin"
)
print(user.id)  # "usr_abc123"

List Users

Retrieve a paginated list of users in your organization. Results are sorted by creation date, newest first.

users = client.users.list(limit=20, offset=0)
for user in users.data:
    print(f"{user.name} ({user.email})")

Update a User

Only the fields provided in the request body will be updated. All other fields remain unchanged.

{
  "name": "Alice Johnson-Smith",
  "role": "superadmin",
  "metadata": {"department": "engineering"}
}

Error Handling

The API returns standard HTTP status codes. Errors include a JSON body with code and message fields:

{
  "error": {
    "code": "invalid_api_key",
    "message": "The API key provided is expired or invalid."
  }
}

Common error codes:

  • 401 Unauthorized — Invalid or missing API key
  • 403 Forbidden — Insufficient permissions for this endpoint
  • 404 Not Found — The requested resource does not exist
  • 429 Too Many Requests — Rate limit exceeded, retry after the Retry-After header
